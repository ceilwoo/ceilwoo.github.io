Firefox这些年是怎么作死的

本D大概差不多10年前就开始用Firefox了，一度是忠实的F粉，各种插件用得不亦乐乎。然而这货越来越作死，3年前已经转变为F黑，投奔Chrome阵营。除了以调试为目的就不再用了:(

简述一下最近几年Firefox是怎么折磨人的：

  1. 2010年的Firefox，想在多台设备上同步配置数据的话需要自己手动操作。然而，Firefox的配置文件夹藏得特别隐秘，你千辛万苦找到它，复制去一个新的Firefox上，却发现不能使用。于是有个叫Foxmarks的书签同步插件诞生了，几乎人手必备，愉快了没多久，突然Foxmarks由于资金问题关闭，然后F粉们又继续过上水深火热的生活。 （Foxmarks后来改名叫Xmarks复活了，不过现在没什么使用的必要）
  2. 2012年的Firefox。比同行晚了几年终于捣鼓出自家的数据同步功能了。当时Chrome登陆个谷爹账号就可以同步书签等资料了，Firefox是什么情况呢？你除了要申请个账号之外，每台机的还会有1串超长的唯一识别码，这个超长识别码还需要搭配一个8位的安全码，缺一不可。也就是说你要完成同步资料的操作，除了需要记住你的F账号密码，还要记住一个超长的字符串和一个不是很短的密码(官方说这样才安全)。有次重装系统时不慎把后面两个东西弄丢了，然后就没有然后了!
  3. 2013年的Firefox。使用时资源占用越来越大，有时开10多个tab就卡成狗。这个问题刚开始出现的时候我就忍了，开少点tab就是了，毕竟是F粉。但是还有个致命问题，关闭了浏览器，下次打开时tab会有一定的几率丢失，以每天开2次的Firefox的频率来计算，一年大概会丢失个四五次，对于一个把tab当书签用的人来说，丢一次就跟短期失了忆一样。（最近的版本有所好转）
  4. Firefox Hello。N年前更新多了一个这玩意，而且还是标配，不能删除。你说一个好好的浏览器加个聊天功能这是个什么情怀？好吧，既然不能删除我们就Disable掉吧，坑爹的是关掉之后，后台还是会一直在发hello数据包。于是你在调试的时候，时不时就会跑出一个hello的请求出来。有时你写了一阵子代码，切回来一看，发现整个屏幕都是hello,hello,hello
  5. 2016年的Firefox。48.0.1版本。最近还在用firefox的原因的是插件在同行里边能实现更多底层的功能，譬如大名鼎鼎的Firebug、Charles之流，简直是瑞士军刀般的存在，调试的时候都习惯用上。然后最近的更新呢。( In Firefox version 43 and above, Firefox prevents you from installing unsigned add-ons and disables any unsigned add-ons that are already installed. ) 43版本是警告，48直接就不让用了。搞个小插件都得先到Firefox那签名认证下。官方的解决方案呢，让你去找其他替代品或者找作者开发个签名版本，呵呵。

虽然想说走好不送，但是总舍不得Firebug这插件，感觉就像是被狗日了。
